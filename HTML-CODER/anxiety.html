Самый хороший и лучший способ был описан в сайте учебного курса. И лучше не тащить всю кучу постов, а конвертировать в json только один точечный пост.

https://www.mousedc.ru/learning/65-sql-select-where-like-and-or/

SELECT * FROM `USERS` WHERE `NAME` LIKE 'Мышь%';

В таком случае отображение поста отобразится по имени “NAME”, а если искать по айдишнику, то по этому принципу ‘id’ LIKE ‘%170%’ отобразит строку всей записи в базе с ключом по айдишнику.

Хоть яваскрипт и так справляется на отлично с перебором многомерного массива из объекта переданного с помощью php в формате json. Но если распределить баланс по нагрузке, выйдет что проще делать выборку на сервере. Таким образом кнопка < button> играет роль щупа или пальца которым можно потыкать базу данных через пиашпи и незамедлительно получить горячий отклик через аякс.
